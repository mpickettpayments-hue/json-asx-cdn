User Guide

Welcome to XJSON, a creator-first OS where apps are described in JSON. This guide covers the core flow: explore, edit, preview, and publish.

1. Explore the OS

  • Home: Featured apps and quick actions.
  • Store: Browse installable examples and templates.
  • Library: Your installed or remixed apps.
  • Downloads: (coming soon) Queue for packaged assets.
  • Settings: Configure Admin PAT and LangFlow endpoint.

2. Channels

Switch between latest, stable, and nightly datasets using the top-left chips. This controls which OS dataset is loaded.

3. Edit in the Fullscreen Editor

  1. Open the Editor (sidebar → code icon or Home → Open Editor).
  2. Select a JSON app from the dropdown.
  3. Modify the JSON in the Editor tab.
  4. Switch to Preview to see live changes (auto-refresh).

4. Publish to Your Subdomain

  1. Go to Settings and paste a repo-scoped GitHub PAT (Contents: Read/Write).
  2. Open the Editor, choose the app, click Publish.
  3. Apps publish to /users/<uid>/apps/<id>/app.json, and your subdomain is https://<uid>.xjson.app/.

5. Provisioning

On first sign-in (main domain), the OS can provision your vault and starter app. It creates:

/users/<uid>/
  manifest.json, profile.json, library.json, ledger.json, vault.json
  apps/starter/app.json

6. AI (Coming Soon)

The Editor includes a placeholder AI tab for LangFlow-based agents that will scaffold scenes, propose remixes, and write files into your app tree when authorized.

7. Troubleshooting

  • Use the Debug button in the top bar to see fetches and errors in real-time.
  • If a channel does not load, check that ./latest/os.json (or stable/nightly) exists and is valid JSON.
  • Hard refresh (Ctrl/Cmd + F5) to bypass caches.

That’s it! Build boldly. If you need deeper templates, XP/badges, or AI bots, they’re next on the roadmap.
